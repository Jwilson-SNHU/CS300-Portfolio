CS 300 Portfolio Reflection

What was the problem you were solving in the projects for this course?

The goal of these projects was to develop software that helps academic advisors manage course information for students. The program needed to load course data from a file, organize it efficiently, display a sorted list of courses, and allow users to search for individual courses and their prerequisites. This required selecting an appropriate data structure and implementing algorithms that produced efficient and accurate results.

How did you approach the problem? Consider why data structures are important to understand.

I began by analyzing different data structures, including vectors, hash tables, and binary search trees. After comparing their run-time performance and memory usage, I chose a binary search tree because it provides efficient searching while naturally maintaining data in sorted order through an in-order traversal. Understanding data structures is important because the choice of structure directly affects the speed, scalability, and maintainability of a program.

How did you overcome any roadblocks you encountered while going through the activities or project?

One of the biggest challenges was reading the input file correctly and handling different file formats. I also encountered issues with loading course data, validating prerequisites, and ensuring the menu handled user input properly. I resolved these issues by testing each function individually, carefully tracing program execution, improving input validation, and making incremental changes until the program behaved correctly.

How has your work on this project expanded your approach to designing software and developing programs?

This project taught me that planning before coding leads to better software design. Creating pseudocode and comparing data structures before implementation helped simplify the coding process. I also gained more experience breaking large problems into smaller functions that each perform a single task, making the overall program easier to develop and debug.

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

This project reinforced the importance of writing modular code with meaningful function names, comments, and clear organization. Separating the program into reusable functions improved readability and made debugging easier. I also learned the importance of validating user input, handling errors gracefully, and designing code that can be expanded with additional features in the future.
